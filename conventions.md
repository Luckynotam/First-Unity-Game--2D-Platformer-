# General Conventions

**Variable Naming**

When naming a variable you should use lowerCamelCase, also names of variables should be consitent and tell what is in the variable. 

**Functions / Classes**

When naming a funcition or Class you should use UpperCamelCase / PascalCase 

When naming Function make sure the name of the function describes what the function does. 

When naming a Class, make sure you name it after what kind of class you are making if you are making a class for a sum name it Sum. If you are defying properties of a Object name it after that object.

Remember also when naming classes PascalCase !!!!!!!

**Use of Acesses Modifier**

When declaring variable, when possible you should make use of [SerializeField] and public accses modifers. 

This makes sure you can approach your variables more easily. 

**Testing**
When making a change, make sure to test it before making a commit.

If the function breaks later, you should fix it in a new sperate commit.
